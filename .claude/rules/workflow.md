# 任务执行流程

本文件定义执行任务的标准流程，确保高质量交付。

## 1. 检索优先

回答问题或执行任务前，必须先全面检索相关代码/文件：
- 以全面性为首要考量
- 穷举一切可能性
- 不遗漏关键信息

## 2. 需求澄清

用户需求通常模糊，需要主动澄清：
- 通过多角度、多维度的提问引导用户明确需求
- 最终确认理解是否正确
- 记录确认后的需求

## 3. 精准定位

基于检索结果和明确需求：
- 精准定位需改动的代码部分
- 不遗漏、不多余
- 评估影响范围

## 4. 信息充分性判断

评估当前信息是否足够：
- 不足时从项目获取更多信息
- 必要时向用户提问
- 循环迭代步骤 1-3 直到信息充分

## 5. 方案讲解

对修改计划进行详略得当的讲解：
- 善用伪代码辅助说明
- 说明关键决策点
- 提示潜在风险

## 关键检查点

### 任务开始
- 回显关键约束
- 确认工具可用性与降级方案

### 编码前
- 完成规划
- 理解现有代码

### 实施中
- 遵循质量标准
- 记录重要决策

### 完成后
- 验证功能与静态检查
- 更新测试与文档
- 总结关键约束与最佳实践

## Git 提交规范

提交信息格式：`<emoji> <type>: <description>`

| Emoji | 类型 | 说明 |
|-------|------|------|
| ✨ | New Features | 新功能 |
| 🐞 | Bug Fixes | BUG修复 |
| 🔨 | Dependency Upgrades | 依赖升级 |
| 📔 | Documentation | 更新文档 |
| ♻️ | Refactor | 代码重构 |
